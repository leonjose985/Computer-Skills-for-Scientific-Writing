# Цель работы

Изучить структуру и правильный способ размещения изображений в научных работах, приобрести навыки и умения в объективном и нейтральном написании формальных исследований с использованием средств LaTeX для работы с плавающими объектами, перекрёстными ссылками и параметрами размещения изображений.

# Ход выполнения

## Включение пользовательских изображений в документ

На первом этапе было выполнено включение собственных изображений в документ LaTeX взамен стандартных демонстрационных файлов. Для этого использовалась команда `\includegraphics` с указанием пути к файлу изображения. Результат демонстрирует корректное отображение пользовательского изображения в теле документа.

![Результат отображения пользовательского изображения в PDF](image/02.png){ #fig:002 width=70% }

## Параметры масштабирования: height, width, angle и scale

Были изучены основные параметры команды `\includegraphics`: `height` для задания высоты, `width` для задания ширины, `angle` для поворота изображения и `scale` для масштабирования относительно исходного размера. Каждый параметр влияет на итоговое отображение изображения независимым образом.

![Исходный код с параметрами height, width, angle и scale](image/03.png){ #fig:003 width=70% }

![Результат применения различных параметров масштабирования](image/04.png){ #fig:004 width=70% }

## Размер изображения относительно ширины текста и строки

С помощью параметра `width` было задано два варианта размера изображения: относительно ширины текстового блока (`\textwidth`) и относительно ширины строки (`\linewidth`). Было проверено поведение обоих вариантов при использовании опции двух колонок документа и без неё. В однocolоночном режиме оба параметра дают одинаковый результат, тогда как в двухколоночном режиме `\linewidth` учитывает ширину текущей колонки.

![Сравнение textwidth и linewidth в одноколоночном режиме](image/05.png){ #fig:005 width=70% }

![Сравнение textwidth и linewidth в двухколоночном режиме](image/06.png){ #fig:006 width=70% }

## Плавающие объекты и спецификаторы положения

С использованием пакета **lipsum** был сформирован достаточно длинный текст для демонстрации поведения плавающих объектов. Были опробованы спецификаторы положения `h` (здесь), `t` (вверху страницы), `b` (внизу страницы), `p` (на отдельной странице) и `H` (принудительно здесь, требует пакет `float`). Было установлено, что спецификаторы взаимодействуют с алгоритмом размещения LaTeX: при недостатке места спецификатор `h` автоматически заменяется на `t` или `b`.

![Исходный код с различными спецификаторами положения](image/07.png){ #fig:007 width=70% }

![Результат размещения плавающих объектов с разными спецификаторами](image/08.png){ #fig:008 width=70% }

## Перекрёстные ссылки: метки и количество запусков компиляции

Были добавлены новые разделы, подразделы и нумерованные списки в тестовый документ с использованием команд `\label{}` и `\ref{}`. Было установлено, что для корректной работы перекрёстных ссылок требуется **два запуска** компиляции `pdflatex`: первый запуск формирует файл `.aux` с метками, второй — подставляет корректные номера в ссылки.

![Поведение меток при первом и втором запуске компиляции](image/09.png){ #fig:009 width=70% }

## Положение метки относительно подписи и уравнений

Было проверено поведение команды `\label{}` при её размещении до и после команды `\caption{}` в плавающих объектах. Установлено, что метка должна располагаться **после** `\caption{}`, иначе ссылка указывает на предыдущий раздел, а не на сам объект. Аналогично была проверена метка уравнения, помещённая после закрывающей команды окружения: в этом случае ссылка также формируется некорректно.

![Результат при размещении метки до и после подписи](image/10.png){ #fig:010 width=70% }

# Выводы

В ходе выполнения лабораторной работы были приобретены необходимые знания и навыки для работы с изображениями и ссылками в документах LaTeX:

- включение пользовательских изображений с помощью команды `\includegraphics` и управление их размером с использованием параметров `height`, `width`, `angle` и `scale`;
- понимание разницы между `\textwidth` и `\linewidth` при задании ширины изображений, в том числе в двухколоночном режиме;
- работа с плавающими объектами и спецификаторами положения `h`, `t`, `b`, `p` и `H`, а также понимание алгоритма их взаимодействия;
- использование команд `\label{}` и `\ref{}` для создания перекрёстных ссылок и понимание необходимости двух запусков компиляции для их корректной работы;
- установление правильного порядка размещения команды `\label{}` относительно `\caption{}` в плавающих объектах и относительно закрывающей команды в окружениях уравнений.

Полученные знания позволяют грамотно оформлять изображения и ссылки в научных работах формата `.tex`, обеспечивая профессиональный вид и согласованность документа.
